Суть проєкту ecommerce-api — це створення потужного бекенду для інтернет-магазину. Ми закладали його як сучасний, масштабований REST API, який повинен керувати всіма основними процесами онлайн-комерції.

Ось ключові напрямки, які ми планували або вже почали реалізовувати:

1. Керування товарами (Product Management)
Створення каталогу товарів з категоріями.

Зберігання інформації про ціни, опис та наявність на складі.

Пошук та фільтрація продуктів.

2. Користувачі та безпека (Auth)
Реєстрація та автентифікація (зазвичай за допомогою JWT-токенів).

Розподіл ролей: клієнт (може купувати) та адміністратор (може додавати товари та змінювати замовлення).

3. Кошик та замовлення (Orders & Cart)
Логіка додавання товарів у кошик.

Оформлення замовлення, розрахунок загальної вартості.

Зміна статусів замовлення (наприклад: "В обробці", "Відправлено", "Завершено").

4. Технічний стек (Твоя база)
Python як основна мова.

Django FastAPI.

PostgreSQL як основна база даних.

Docker для того, щоб усе це легко запускалося в будь-якому оточенні (враховуючи твій перехід на WSL/Ubuntu).

🚀 Проєкт: AI-Powered eCommerce API
Суть проєкту: Створення сучасного бекенду для інтернет-магазину, де традиційна комерція (товари, замовлення) підсилюється штучним інтелектом (LLM) для персоналізації, розумного пошуку або автоматичного опису товарів.



Технологія,Роль у проєкті
Python 3.12+,Основна мова розробки.
FastAPI,Асинхронний фреймворк для створення швидких ендпоінтів.
Poetry,Керування залежностями (замість pip/requirements.txt).
PostgreSQL,Надійна база даних для товарів та користувачів.
LangChain / OpenAI SDK,Інтеграція з LLM (ChatGPT або аналоги).
GitHub Actions,Автоматизація: перевірка коду (CI) при кожному пуші.
Docker,Контейнеризація для легкого розгортання будь-де.



📝 Етапи розробки (Roadmap)
Етап 1: Налаштування "Фундаменту"
Ініціалізація проєкту через poetry init.

Налаштування інтерпретатора в PyCharm (через WSL).

Створення базового FastAPI додатку (main.py) з одним тестовим маршрутом.

Етап 2: Автоматизація (CI)
Створення .github/workflows/ci.yml.

Налаштування автоматичної перевірки: чи встановлюються залежності та чи немає критичних помилок у коді.

Етап 3: База даних та Моделі
Підключення бази через SQLAlchemy або Tortoise ORM.

Створення моделей: User, Product, Order.

Етап 4: Інтеграція LLM
Додавання функціоналу AI. Наприклад: "Згенеруй опис товару за назвою" або "Порадь товар на основі уподобань".

Етап 5: Docker & Деплой
Написання Dockerfile та docker-compose.yml.

Запуск усієї інфраструктури однією командою.
